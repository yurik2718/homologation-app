## Minor / Guardian (Variant 1.5)

Students can be minors (parents pay) or adults (pay themselves).

| Field on `users` | Purpose |
|---|---|
| `is_minor` | `true` if under 18 |
| `guardian_name` | Parent name |
| `guardian_email` | Parent email (Stripe invoices go here) |
| `guardian_phone` | Parent phone |
| `guardian_whatsapp` | Parent WhatsApp (AmoCRM uses this if minor) |
| `guardian_user_id` | Optional FK → users (future: parent login) |

**MVP:** guardian fields are text, parent gets invoices/notifications by email.
**Future:** parent registers, links to child via `guardian_user_id`, can log in and see status.
